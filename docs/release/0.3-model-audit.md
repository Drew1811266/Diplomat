# Diplomat 0.3.0 Model Audit

This audit covers the curated open-source model registry for Diplomat 0.3.0.

## Registry Scope

The 0.3 model manager exposes only built-in registry entries. Users cannot add arbitrary models from the desktop UI.

| Model ID | Task | Runtime | License | Release Status |
| --- | --- | --- | --- | --- |

| `asr.faster-whisper.medium` | ASR | faster-whisper | MIT | Pinned Hugging Face snapshot manifest |

| `translation.opus-mt.en-zh` | Translation | ct2-marian | Apache-2.0 | Pinned CTranslate2 snapshot manifest |
| `translation.qwen3.4b` | Translation | local-llm | Apache-2.0 | Pinned Hugging Face snapshot manifest |

## Required Evidence

Before Diplomat 0.3.0 can be accepted:

- Every registry `source_url` must point to a pinned `hf://repo@revision` snapshot source.
- Every registry checksum must be a real SHA256 checksum for the exact pinned snapshot manifest.
- Every license name and license URL must be present.
- Any attribution obligations must be reflected in release notes or bundled notices.

## Current Gate

The release readiness API reports model registry blockers when placeholder checksums or bare repository URLs remain. The 0.3.0 registry now uses pinned Hugging Face snapshot sources and manifest checksums; future model changes must preserve that gate.
