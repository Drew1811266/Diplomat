# Diplomat 0.3.0 Acceptance Script

This script is the release manager checklist for accepting Diplomat 0.3.0. Do not tag or publish a build while any blocker remains in the release readiness report.

## Preconditions

- Work from `main` after the v0.30 stage branch has been merged.
- Confirm `git status --short` is empty.
- Use Windows for the desktop packaging pass.
- Use an operator-provided local video file for media smoke tests. Do not rely on bundled sample projects.

## Automated Checks

Run the repository checks from the workspace root:

```powershell
$env:DIPLOMAT_FFMPEG_PATH = (Resolve-Path ".\.dev\tools\ffmpeg-release-essentials\ffmpeg-8.1.1-essentials_build\bin\ffmpeg.exe").Path
$env:DIPLOMAT_FFPROBE_PATH = (Resolve-Path ".\.dev\tools\ffmpeg-release-essentials\ffmpeg-8.1.1-essentials_build\bin\ffprobe.exe").Path
node .\scripts\verify-version.mjs
node .\scripts\verify-release-assets.mjs
corepack pnpm -r test
corepack pnpm -r typecheck
python -m pip install -e .\worker[dev]
python -m pytest
```

If using the consolidated PowerShell check script, first confirm `corepack pnpm --version` works, then run:

```powershell
.\scripts\check.ps1
```

## Desktop Smoke

1. Build the web app and Tauri desktop shell.
2. Start Diplomat Desktop.
3. Open Settings and confirm Worker, FFmpeg, FFprobe, data, model, and log paths render.
4. Open Help Center from the application rail and confirm English and Chinese content is available.
5. Open Project Center, import a local video, and open the Workbench.
6. Run analysis with an installed curated ASR model.
7. Run translation when a curated translation model is installed and supports the selected language pair.
8. Save edits, export subtitle files, and render a burned-in video export.
9. Confirm export output plays and contains visible subtitles.

## Release Readiness Gate

Before accepting Diplomat 0.3.0:

- Open Settings and confirm the release readiness panel reports `Release ready`.
- Query `GET /release/readiness` and confirm `ready` is `true`.
- Confirm `summary.blocker` is `0`.
- Confirm the readiness report names the FFmpeg and FFprobe paths used for the release runtime.
- Attach the readiness JSON to the stage gate note.

## Acceptance Rule

Diplomat 0.3.0 is accepted only after every automated check passes, the desktop smoke pass is complete, and the release readiness gate has zero blockers.
