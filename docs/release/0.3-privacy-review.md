# Diplomat 0.3.0 Privacy Review

This review records the privacy expectations for Diplomat 0.3.0.

## Default Data Flow

- Imported videos remain on the local machine.
- Project metadata, subtitles, drafts, snapshots, logs, cache, models, and exports are stored under the local Diplomat runtime directories.
- ASR, translation, subtitle editing, validation, subtitle export, and burned-in video export are designed to run locally for the formal 0.3 workflow.
- Remote ASR or translation services are not part of the formal 0.3 user interface.

## Model Downloads

- Users can only install curated open-source models exposed by the built-in model manager.
- Each model entry must include a license, source URL, SHA256 checksum, and terms summary before final release acceptance.
- Downloads must pass checksum verification before the model is treated as installed.

## Logs And Diagnostics

- Worker stdout and stderr logs are local files.
- Logs can include local file paths and task errors.
- Logs should not be uploaded automatically.

## User-Facing Requirements

- Settings must show runtime paths so users can inspect local storage.
- Help Center must state that the default production workflow is local.
- Release readiness must block the release if model audit, FFmpeg, packaging, or documentation checks fail.

## 0.3.0 Decision

Diplomat 0.3.0 is acceptable from a privacy perspective only when release readiness reports zero blockers and the formal workflow remains local by default.
