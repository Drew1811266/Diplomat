# Diplomat 0.3.0 FFmpeg Audit

This audit covers FFmpeg and FFprobe requirements for Diplomat 0.3.0 subtitle and burned-in video export.

## Runtime Requirement

Diplomat uses FFmpeg tooling for media inspection, subtitle export validation, and burned-in video rendering. A release-ready Windows installation must provide:

- `ffmpeg`
- `ffprobe`

The desktop Settings page and Worker release readiness endpoint both surface tool availability.

For the 0.3.0 validation run in this workspace, the approved local runtime paths are:

```powershell
$env:DIPLOMAT_FFMPEG_PATH = (Resolve-Path ".\.dev\tools\ffmpeg-release-essentials\ffmpeg-8.1.1-essentials_build\bin\ffmpeg.exe").Path
$env:DIPLOMAT_FFPROBE_PATH = (Resolve-Path ".\.dev\tools\ffmpeg-release-essentials\ffmpeg-8.1.1-essentials_build\bin\ffprobe.exe").Path
```

Final installers must either carry equivalent release-approved binaries or set equivalent runtime paths before the Worker starts.

## Acceptance Checks

Before accepting Diplomat 0.3.0:

- Confirm `ffmpeg -version` exits successfully in the release runtime.
- Confirm `ffprobe -version` exits successfully in the release runtime.
- Confirm the app can render a burned-in MP4 from an operator-provided local video.
- Confirm the output file contains H.264 video, playable audio when the source has audio, and visible subtitles.
- Record the FFmpeg build source and license posture used for the release package.

## Blocking Conditions

The release is blocked when:

- FFmpeg is missing.
- FFprobe is missing.
- FFmpeg or FFprobe exits with an error.
- The burned-in export smoke test fails.
- The release package cannot explain how the FFmpeg runtime is supplied to users.

## 0.3.0 Decision

Diplomat 0.3.0 can only be accepted after FFmpeg and FFprobe are available in the release runtime and the burned-in video export smoke test passes.
